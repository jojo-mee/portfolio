OVERVIEW:

CONTEXT

This website is the official sales website of a record/album company. The company sells records and albums including CDs and vinyls, and offers online shopping and delivery services. The main potential customers of this company are music lovers, including fans of singers or idols and vinyl collectors. The company aims to provide consumers with the most complete selection of goods and satisfy their demand for music as much as possible. █

CHALLENGE & SOLUTION:

Our challenge in this section is the responsive design. We found it difficult to make buttons and fonts sizes look nice and harmonious on screens of different devices.

STEP 3: Create Web Page

The beginning of coding part, follow the wireframe mockup previously created and fill the website with real content (text and images).

In the home page, we found it contain too many texts and did not show the hierarchy of the content.

We only have one buttom to add item into shopping cart. This is very inconvenience.

From the feedback, the checkout page seems messy and hard to read.

There is no direct path to shooping cart.

CHALLENGE & SOLUTION:

After we create the first draft of the web page through the code, we found that many details related to user experience were not well thought out. Most of these issues will only become apparent after use/go through, so don't be afraid to produce a planned piece of content. Only by producing samples can you get better feedback and improve it.

STEP 4: Finalize Web Page

Polish the websites. Fix all the bugs and adjust responsive setting to make the website looks perfect on different devices.

We add more image to balance the content, and use icon to help user understand.

We add a buttom to create better user experience. There is also visual beedback on buttoms.

We reorganize the checkout page to bring better user experience.

We add a path directly guide user to all the primary page.

TAKEAWAYS:

Functional and Responsive Design

When creating a website, it is important that it has an attractive look and feel, but it is also important that it is functional and adaptable to various operating systems. It is essential to create a universal and responsive design that takes into the diversity of users and their different needs. In another word, we have to provide a website that can responses and functions on all different platforms. █

Design and Code

We used a style guide to help the team maintain consistency when coding our design into a web. This allows for a better common reference among the team, and the upfront effort saves a lot of time in later process. At the same time, we moved the design to the site itself as soon as possible. A responsive design doesn't look as same on a flat surface, so continuing the design process through the style guide is a much better option. █
